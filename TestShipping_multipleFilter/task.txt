В БД есть таблица следующей структуры:

1. Идентификатор записи
2. Номер месяца
3. Год
4. Город отправления
5. Город назначения
6. Наименование заказчика
7. Наименование компании перевозчика
8. Тонны плановых перевозок  
9. Тонны фактических перевозок 
10 - 41 Фактические перевозки в тоннах за каждый день месяца 
(если число дней месяца меньше 31, то "лишние" дни содержат значение null). Отчет может быть сформирован "посреди" месяца. 
В этом случае по "будущим" дням значение 0

В таблице за один месяц может быть несколько сотен строк по разным направлениям и разным перевозчикам

Число городов, заказчиков и число перевозчиков - по 20 каждой категории.
Возможны "встречные" направления 
(Одна строка из города А в город Б. А другая строка из города Б в город А)

Ваша задача

Создать простейшую БД с одной этой таблицей (нормализовать не нужно - города и перевозчики хранятся в текстовом виде) - MS SQL

Подготовить "тестовые" данные за 2 месяца

Реализовать один веб-интерфейс для работы с данными из этой таблицы в виде табличного отчета (плюсом будет организация страничного отображения)

На странице должна быть возможность выбора за какой месяц мне смотреть данные

Также необходимо наличие множественного выбора для фильтра по 
	- городам отправления и 
	- городам назначения, 
	- закзачику и 
	- перевозчику

Плюсом будет возможность выгрузить отчет в виде XLS-файла
